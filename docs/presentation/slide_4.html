Clean Multi-Tier Architecture Powers the API

The system follows a standard multi-tier architecture. HTTP requests are routed by FastAPI, validated by Pydantic schemas, processed through SQLAlchemy ORM queries, and persisted in a SQLite database. This separation of concerns ensures that each layer can be tested, modified, or replaced independently.

Routing Layer
FastAPI
Handles 25 endpoint definitions across 7 groups, managing HTTP requests and responses.
Validation Layer
Pydantic
Enforces rating ranges (1-5), string lengths, and required fields before processing.
Data Access Layer
SQLAlchemy ORM
Manages complex joins and aggregations, mapping Python objects to database tables.
Database
SQLite
Provides zero-configuration persistence with 25 seeded tracks injected on startup.
