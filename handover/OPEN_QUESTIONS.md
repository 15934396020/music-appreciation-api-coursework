# Open Questions

## Current Open Questions

| Topic | Question | Current Recommendation |
|---|---|---|
| Seed dataset | Should the project import a real subset from FMA immediately, or use a manually curated starter CSV first? | Start with a small curated CSV, then expand if time allows |
| Authentication | Is authentication necessary for the first milestone? | No, postpone until core CRUD is stable |
| Deployment | Should the first GitHub version focus on source code only or also include cloud deployment instructions? | Source code first, deployment instructions second |
| External APIs | Should MusicBrainz be integrated in the first version? | No, keep as future extension only |
| Data scale | How many tracks should be seeded initially? | Begin with about 20 to 50 tracks for development, then expand |

## Current Risks

| Risk | Impact | Mitigation |
|---|---|---|
| Over-engineering too early | Delays core completion | Build a small working baseline first |
| Missing handover updates | Causes repeated work across accounts | Update handover files every session |
| Delayed documentation | Harder final submission preparation | Maintain README and API plan from the start |
| Too much focus on data sourcing | Slows API implementation | Use a small seed dataset first |
