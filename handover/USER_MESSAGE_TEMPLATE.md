# User Message Template for a New Account

## When GitHub Repository Is Available

The user can send the following message together with the repository link:

> Please continue this coursework project from the existing GitHub repository. Read `handover/FOR_NEXT_ACCOUNT.md` first, then `handover/CURRENT_STATUS.md` and `handover/NEXT_ACTIONS.md`. Do not redesign the project unless necessary. Continue from the recorded next tasks, update all handover files before stopping, and keep the repository ready for the next account.

## When Only the Zip File Is Available

The user can send the following message together with the zip file:

> Please continue this coursework project from the attached project archive. Read `handover/FOR_NEXT_ACCOUNT.md` first, then `handover/CURRENT_STATUS.md` and `handover/NEXT_ACTIONS.md`. Do not redesign the project unless necessary. Continue from the recorded next tasks, update all handover files before stopping, and keep the repository ready for the next account.

## Minimal Version

If the user wants an even shorter version, the following is enough:

> Please read `handover/FOR_NEXT_ACCOUNT.md` first and continue from `handover/NEXT_ACTIONS.md`.

## What the User Usually Does Not Need to Repeat

Once the repository or zip file is provided, the user usually does not need to repeat these details because they are already documented in the repository:

| Detail | Already documented in |
|---|---|
| Project theme | `handover/FOR_NEXT_ACCOUNT.md` |
| Coursework target level | `handover/FOR_NEXT_ACCOUNT.md` |
| Current progress | `handover/CURRENT_STATUS.md` |
| Immediate next tasks | `handover/NEXT_ACTIONS.md` |
| Work history | `handover/SESSION_LOG.md` |
| Open issues | `handover/OPEN_QUESTIONS.md` |

## Recommendation

If possible, always prefer sending the GitHub repository link instead of only a chat summary, because the new account can then inspect both the code and the handover documents directly.
