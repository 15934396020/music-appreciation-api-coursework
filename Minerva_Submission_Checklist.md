# Minerva Submission Checklist for XJCO3011 Coursework 1

Based on the assessment brief for the "Music Appreciation and Discovery API" project, the following materials must be prepared and submitted via Minerva by the deadline of **21 April 2026**.

## 1. Technical Report (PDF)
This is the primary document to be uploaded to Minerva. It must contain the following components:
- A direct link to your public GitHub repository.
- A direct link to your API documentation (which should also be accessible through the GitHub repository).
- A direct link to your presentation slides and any other visual aids (hosted on a platform such as Google Drive or OneDrive).
- A concise written report (maximum 5 pages) outlining key design and architectural choices.
- Justification for the chosen technology stack (FastAPI, SQLAlchemy, SQLite).
- Reflection on challenges faced, the testing approach taken, and lessons learned during development.
- Discussion of current limitations and potential areas for future improvement.
- A Generative AI declaration and thoughtful analysis of its usage, included as an appendix.

## 2. Presentation Slides (PPTX)
The slides used for your 5-minute oral presentation must be submitted. They must adhere to academic presentation quality guidelines and include dedicated sections covering:
- Version control practices and a review of the commit history.
- An overview of the API documentation.
- Highlights from the technical report.
- A summary of all deliverables (code repository, API docs, technical report).

## 3. GitHub Repository Requirements
While the repository itself is not directly uploaded to Minerva, the link provided in the Technical Report will be evaluated. The repository must be public and contain:
- Versioned source code with a clear and consistent commit history.
- A comprehensive `README.md` file describing setup instructions and providing a project overview.
- The API documentation in PDF format.
- Fully runnable code that corresponds exactly to the version presented during the oral examination.

## Important Reminders
Failure to include the API documentation, the technical report, the GenAI declaration, version control history, a `README.md`, or submitting non-runnable code will result in the corresponding component being graded as a Fail.

Ensure all links provided in the Technical Report are publicly accessible or accessible to the examiners. The oral presentations are scheduled to take place during the week commencing 28 April 2026 through 7 May 2026. You must arrive 10 minutes before your scheduled start time with all devices, program code, and presentation materials ready.
